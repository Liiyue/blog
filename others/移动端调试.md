作为移动端开发er,深受各种机型兼容性的困扰，如何进行移动端调试显的相当重要，因此特地总结了各种移动端调试方法。
## 1. Andriod调试
### 1.1 koala webview
#### what you need
+ 开发计算机上已经安装chrome32或更高版本
+ 一根可以将您的 Android 设备连接至开发计算机的 USB 数据线
+ Android 4.0 或更高版本
+ Koala app 下一个～
#### how to debug
+ Stpe1: 启用手机的’USB调试模式’
  一般是在 `设置 > 系统 > 开发人员选项  > USB调试` 中开启本模式。不同手机会有细微的差别，`开发人员选项`有可能默认处于隐藏状态，具体如何操作，还请百度。
+ Step2: 开启考拉app的webview调试开关
  `打开考拉app > 最下方tab 进入'我的考拉' > 点击右上方'设置'按钮进入设置页面 > 点击'关于考拉' 进入关于考拉页面 > 返回设置页面 > 点击最上方'设置'tab, 连击10下，弹出debug弹窗 > 输入'2016-818'进入debug模式 > 打开webview调试开关` 
+ Step3: 将Android设备连接至计算机，选择连接方式为传输文件，并允许USB调试；
+ Step4: 在考拉app中打开所需要调试的H5页面；
+ Step5: 在开发计算机打开chrome，并进入`chrome://inspect`页面，点击对应的inspect即可进入调试页面啦～
  ![image.png](https://upload-images.jianshu.io/upload_images/1860854-2304fedae7f15bc2.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

![image.png](https://upload-images.jianshu.io/upload_images/1860854-da0f282dba8fd5f5.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)

### 1.2 浏览器 -- 以手机chrome为例
#### what you need
+ 开发计算机上已经安装chrome32或更高版本
+ 一根可以将您的 Android 设备连接至开发计算机的 USB 数据线
+ Android 4.0 或更高版本
+ 手机chrome浏览器下一个～
#### how to debug
+ Step1: 启用手机'USB调试模式'；
+ Step2: 将Android设备连接至计算机，并允许USB调试；
+ Step3: 在Android设备上的chrome浏览器中打开待调试的页面；
+ Step4: 在开发计算机打开chrome浏览器，并进入`chrome://inspect`页面，点击对应的inspect即可进入调试页面啦～

#### Attetion

+ 使用chrome的inspect需要翻墙。。。
+ mac通过转接线可能无法识别andriod机，最好直连



## 2. IOS
### 2.1 kaola webview
#### what you need
+ mac
+ xcode(apple store即可免费下载)
+ 模拟器调试包([戳这里，下载调试包](http://doc.hz.netease.com/pages/viewpage.action?pageId=138907291))
#### how to debug
+ Step1: 打开xcode，并运行模拟器
+ Step2: 安装测试包
+ Step3: 打开考拉app
+ Step4: 打开所需调试的h5页面
  `连续点击模拟器上方的‘大力水手胳膊’图标三次 > 跳转指定h5 > 输入需调试的h5页面url > 确定`
  ![image.png](https://upload-images.jianshu.io/upload_images/1860854-39b4232965b89a3b.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)
+ Step5: 调试对应的h5页面
  ` 打开Safari浏览器 > 开发 > 选择对应模拟器 > 待调试页面url > 点击打开调试器`
  ![image.png](https://upload-images.jianshu.io/upload_images/1860854-0e39964d4b899170.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)
  如果Safari浏览器顶部tab选项中没有`开发`选项，可进行如下设置：
  `Safari浏览器 > 偏好设置 > 高级 > 在菜单栏中显示'开发'菜单 > 勾选 `
  ![image.png](https://upload-images.jianshu.io/upload_images/1860854-79e20f3663fb3025.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)
  如果模拟器木有被Safari浏览器识别，即'开发'下拉菜单中没有对应的模拟器选项，可推出浏览器，重启即可。
### 2.2 safari
#### what you need
+ mac
+ iphone
#### how to debug
+ Step1: 开启设备中相应调试功能
  [ios终端]: ` 设置 > Safari > 高级 > Web 检查器 > 开`
  [mac]: `Safari > 偏好设置 > 高级 > 在菜单栏中显示“开发”菜单 >  勾选`
+ Step2: 连接移动设备与计算机
+ Step3: 在移动设备Safari浏览器中打开所需调试的页面
+ Step4: 在开发计算机Safari浏览器中，`开发 > 对应真机 > 待调试页面url > 点击打开调试器`
  ![image.png](https://upload-images.jianshu.io/upload_images/1860854-8fabf087a0333520.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/1240)
